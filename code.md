    1).接受从发送端来的识别过后的缺陷图片和正常图片，要进行分类，记录信息，同时可以做日志管理

# PLC 缺陷检测系统开发设计文档

---

# 1. 项目概述

## 1.1 项目背景

为了实现 PLC 产品生产过程中的自动化缺陷检测，设计一套基于深度学习视觉检测的跨平台检测系统。

系统采用 **Windows 发送端 + macOS 接收端** 的架构。

Windows 端负责摄像头采集、AI 模型推理及检测结果上传；macOS 端负责检测结果接收、图片管理、日志管理、数据统计及 Web 可视化展示。

系统仅负责**模型部署与推理**，模型训练工作已完成，运行过程中直接调用 `models` 目录下的 `best.pt` 模型。

---

# 2. 系统总体架构

系统采用 **客户端/服务器（C/S）架构**，整体由三个部分组成：

## （1）Windows 发送端

负责整个视觉检测流程，包括：

* 调用摄像头实时采集 PLC 图像
* 加载 `models/best.pt` 模型进行缺陷检测
* 获取检测类别、置信度等信息
* 保存正常图片及缺陷图片
* 通过 HTTP RESTful API 将检测结果上传至服务器
* 记录本地运行日志

发送端部署于具有 NVIDIA GPU 的 Windows 主机，利用 CUDA 加速模型推理，提高检测速度。

---

## （2）macOS 接收端

负责服务器业务处理，包括：

* 提供 RESTful API 接口
* 接收发送端上传的数据
* 分类保存检测图片
* 将检测信息写入数据库
* 提供日志管理
* 为 Web 管理后台提供数据接口

接收端不参与 AI 推理，仅负责业务管理。

---

## （3）Web 管理后台

Web 管理后台部署于接收端。

主要功能包括：

* 检测记录管理
* 图片查看
* 缺陷统计
* 日志查看
* 设备状态查看

用户通过浏览器即可访问，无需安装客户端。

# 6. RESTful API 设计

## 上传检测结果

```
POST /api/detection-records
```

请求类型：

```
multipart/form-data
```

参数：

| 参数         | 类型       |
| ---------- | -------- |
| image      | File     |
| deviceId   | String   |
| result     | String   |
| defectType | String   |
| confidence | Double   |
| detectTime | DateTime |

返回：

```json
{
    "code":200,
    "message":"success"
}
```

---

## 查询检测记录

```
GET /api/detection-records
```

支持：

* 分页
* 时间筛选
* 设备筛选
* 缺陷类型筛选

---

## 查询单条记录

```
GET /api/detection-records/{id}
```

---

## 删除记录

---

## 获取图片

# 8. 异常处理设计

发送端：

* 摄像头打开失败
* 模型加载失败
* CUDA 不可用
* 网络连接失败
* 上传失败自动重试

接收端：

* 图片保存失败
* 数据库写入失败
* 文件不存在
* REST API 异常
* 磁盘空间不足

# 10. 系统扩展设计

系统预留以下扩展能力：

1. 支持多台 Windows 检测终端同时接入，通过 `deviceId` 区分不同设备。
2. 支持用户登录及权限管理，实现管理员、操作员等角色控制。
3. 支持邮件、企业微信、钉钉等告警通知。
4. 支持模型热更新，在不重新部署系统的情况下替换 `models/best.pt`。
5. 支持图片迁移至 MinIO、NAS 等对象存储。
6. 支持 SQLite 平滑迁移至 MySQL 或 PostgreSQL。
7. 支持检测趋势分析、缺陷统计分析及数据导出。
8. 支持 Docker 容器化部署，提高跨平台部署效率。
